Detailed Description

Handles automatic testing functionality.

The basic approach with SDL_AT is to divide the tests into what are called test suites and test cases. Each test suite should have multiple test cases, each test case can have multiple asserts.

To actually test for conditions within the testcase you check asserts, if the asserts fail the failures will be logged in the testsuite and displayed.

Syntax is similar to OpenGL. An example would be:

int f; // Number failed
SDL_ATinit( "My testsuite" );
SDL_ATbegin( "My first testcase" );
if (!SDL_ATassert( (1+1)==2, "Trying '1+1=2'."))
return; // Implicitly calls SDL_ATend if assert fails
SDL_ATend(); // Finish testcase
SDL_ATbegin( "My second testcase" );
if (!SDL_ATassert( (4/2)==2, "Trying '4/2=2'."))
return; // Implicitly calls SDL_ATend if assert fails
SDL_ATend(); // Finish testcase
f = SDL_ATfinish();
